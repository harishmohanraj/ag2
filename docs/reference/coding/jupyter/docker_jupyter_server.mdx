    (Experimental)
    
    Start a Jupyter kernel gateway server in a Docker container.

<b>Parameters:</b>
| Name | Description |
|--|--|
| `custom_image_name` | Custom image to use.<br/><br/>If this is None, then the bundled image will be built and used.<br/><br/>The default image is based on quay.io/jupyter/docker-stacks-foundation and extended to include jupyter_kernel_gateway<br/><br/>**Type:** `str \| None`<br/><br/>**Default:** None |
| `container_name` | Name of the container to start.<br/><br/>A name will be generated if None.<br/><br/>**Type:** `str \| None`<br/><br/>**Default:** None |
| `auto_remove` | If true the Docker container will be deleted when it is stopped.<br/><br/>**Type:** `bool`<br/><br/>**Default:** True |
| `stop_container` | If true the container will be stopped, either by program exit or using the context manager<br/><br/>**Type:** `bool`<br/><br/>**Default:** True |
| `docker_env` | Extra environment variables to pass to the running Docker container.<br/><br/>**Type:** `dict[str, str]`<br/><br/>**Default:** \{} |
| `token` | Token to use for authentication.<br/><br/>If GenerateToken is used, a random token will be generated.<br/><br/>Empty string will be unauthenticated.<br/><br/>**Type:** `str \| GenerateToken`<br/><br/>**Default:** &lt;autogen.coding.jupyter.docker_jupyter_server.DockerJupyterServer.GenerateToken object> |
